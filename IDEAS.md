# 🌊 API 케미로 만드는 블루오션 앱 아이디어

> 이 카탈로그(711개 API)와 카탈로그 밖의 공공·민간 API를 **조합했을 때 1+1=3이 되는**,
> 아직 한국에 없고, 사람들이 생활 속에서 의존하게 되는 앱 아이디어를 발굴·검증한 리포트입니다.

## 어떻게 만든 목록인가

단순 브레인스토밍이 아니라, 아이디어마다 **적대적 이중 검증**을 거쳤습니다.

1. **발굴** — 6개 관점(하루 루틴 / 돈 / 동네·로컬 / 돌봄 / 의외의 데이터 케미 / 습관·리텐션)으로 원석 아이디어 35개 수집
2. **병합·선별** — 중복 통합, 케미가 약하거나 국민앱이 이미 있는 것 탈락 → 18개
3. **시장 검증** — 아이디어마다 조사원이 *"이미 한국에 존재함을 증명하라"*는 임무로 한국어 웹을 수 회씩 검색 (앱스토어·대기업 기능·정부 앱·스타트업)
4. **실현성 검증** — 아이디어마다 기술 검증가가 *"이 API 조합으로는 구현 불가능함을 증명하라"*는 임무로 API 생존 여부·실제 제공 데이터·개인 키 발급 가능성을 확인

점수의 의미:

- **블루오션 점수 (BO, 0~10)** — 높을수록 같은 가치를 주는 서비스가 없음. *경쟁 카테고리가 있어도 핵심 케미가 비어 있으면 점수가 남습니다.*
- **실현성 점수 (FE, 0~10)** — 높을수록 개인·소규모 팀이 무료 API 키만으로 지금 당장 만들 수 있음.

# 🏆 TIER 1 — 지금 만들어도 되는 것

## 1. 코편한아침 — 비염인의 아침 판정 앱

**BO 6.5 / FE 8 · 타겟: 알레르기 비염·결막염·천식 환자 (유병률 수백만 명, 봄·가을 집중)**

비염인은 아침마다 "오늘 약 먹고 나갈까, 마스크 쓸까"를 감으로 정합니다. 꽃가루 지수는 기상청 사이트 구석에, 미세먼지는 다른 앱에, 일교차는 또 다른 앱에 있고 — 정작 **내 증상이 셋 중 무엇 때문인지는 아무도 알려주지 않습니다.**

**케미:** 꽃가루지수 + PM2.5 + 일교차 세 트리거를 가중 합산해야 "오늘 비염 위험 높음, 약 챙겨요" 판정이 가능합니다. 여기에 사용자의 3초 증상 로그(😊/🤧/🤯)를 누적하면 **"당신은 꽃가루보다 일교차에 반응하는 타입"** 같은 개인 상관분석이 나옵니다. 환경 데이터만 주는 앱도, 기록만 하는 앱도 있지만 둘을 교차시켜 원인을 짚어주는 국내 앱은 없습니다.

| 역할 | API | 비고 |
|------|-----|------|
| 꽃가루 | 기상청 꽃가루농도위험지수 3.0 (data.go.kr/data/15085289) | 무료·자동승인. 수목 3~6월·잡초 8~10월만 제공(계절성) |
| 미세먼지 | 에어코리아 대기오염정보 (data.go.kr) | 개발계정 일 500회 → 일일 브리핑엔 충분 |
| 일교차 | 기상청 단기예보 (data.go.kr) | TMN/TMX로 산출, 일 10,000회 |
| 약 정보 | 식약처 e약은요 (data.go.kr) | OTC 항히스타민제 안내·복용 리마인드 |
| 알림·기록 | Telegram Bot (카탈로그) | 인라인 버튼이 3초 증상 로그 UX에 정확히 대응 |
| 리포트 | QuickChart (카탈로그) | 주간 상관분석 카드 이미지 |

**의존성 루프:** 매일 아침 판정 푸시(습관 앵커) → 증상 로그가 쌓일수록 분석이 정확해지는 **데이터 자산 축적**(갈아타기 비용↑) → 봄·가을 "꽃가루 시즌 개막" 재활성화 트리거.

**검증 결과:** 동일 케미는 해외에만 존재(미국 ZYRTEC AllergyCast, 유럽 klarify — 한국 미지원). 국내 최근접은 숨케어(천식·COPD 특화)로 비염 브리핑·합산 지수가 아님. **주의:** 데이터가 전부 공공 API라 네이버 날씨·미세미세가 '합산 지수'는 쉽게 따라할 수 있음 → **방어력은 증상-환경 대조 분석(개인화 데이터)에서 나와야 함.** 꽃가루 오프시즌(11~2월, 7월)은 2요소 판정으로 축소 — 국립기상과학원 '꽃가루 달력' 정적 데이터로 보완.

**MVP:** 위치 1개 등록 → 매일 판정 푸시 + 이모지 3개 증상 버튼. 30일 뒤 주간 리포트 카드 1장. 1인 개발 한 달 규모.

## 2. 시세밥상 — 물가 연동 저녁밥 추천

**BO 6 / FE 8.5 (전 아이디어 중 실현성 최상위·치명 블로커 0건) · 타겟: 장보기·저녁을 직접 하는 자취생, 신혼·맞벌이**

장바구니 물가는 체감으로 계속 오르는데, 뭐가 지금 싸고 제철인지는 마트에 가서야 압니다. **레시피 앱은 가격을 모르고, 가격 사이트(KAMIS)는 레시피를 모릅니다.**

**케미:** KAMIS 일별 시세에서 '전주 대비 하락률 상위 재료'를 뽑고, 그 재료를 주재료로 쓰는 레시피를 식약처 레시피 DB에서 역검색 → **"삼겹살 9%↓ → 오늘은 김치찜"**. 어느 한쪽 API만으로는 불가능한 추천입니다. 레시피 DB에 영양성분 필드가 내장돼 있어 '싸고+단백질 충분' 필터까지 공짜. QuickChart 가격 추세 그래프가 "지금이 저점" 설득력을 만듭니다.

| 역할 | API | 비고 |
|------|-----|------|
| 시세 | KAMIS 농수산물 유통정보 (kamis.or.kr) | 무료·개인 발급. 일별 소매가+등락률, 축산물 포함 |
| 레시피 | 식약처 조리식품 레시피DB COOKRCP01 | 약 1,100건, 재료·열량·영양성분 내장 |
| 차트 | QuickChart (카탈로그) | 가격 추이 이미지를 알림에 첨부 |
| 알림 | Telegram Bot (카탈로그) | 오후 5시 브로드캐스트로 시작 |

**의존성 루프:** 매일 오후 5시 "오늘의 하락 재료 TOP3 + 레시피" 푸시가 퇴근길 장보기 타이밍과 정확히 겹침 → "오늘 안 사면 다시 오른다" 손실회피 + **"이번 달 23,400원 아낌" 누적 절약액** 보상.

**검증 결과:** 동일 케미의 상용 서비스 없음. 만개의레시피는 냉장고 재료 기반이고 가격을 모르며, KAMIS·참가격은 레시피가 없고, 이마트 앱은 특가와 레시피가 서로 연동 안 됨. **주의:** 만개의레시피가 데이터·트래픽을 모두 보유해 기능 하나로 모방 가능한 구조적 위협. GitHub에 유사 발상의 사이드프로젝트(VeggieMeal) 흔적 → 발상 자체는 희소하지 않으니 **속도가 중요.** 구현 팁: KAMIS 품목명↔레시피 재료명 매핑 테이블은 초기 1회 수작업, 주말·공휴일은 시세 데이터 없음(금요일가 기준 처리).

**MVP:** 상위 품목 50개 × 레시피 1,100건 매핑 → 매일 크론 → 텔레그램 발송. 개인화 없이 브로드캐스트로 시작, 1인 개발 3~4주.

# 🥈 TIER 2 — 유망하지만 조건이 붙는 것

## 3. 살림타이밍 — 집안일 골든타임 알림

**BO 5 / FE 8 · 타겟: 자취생·주부·맞벌이 1~2인 가구**

환기하려고 창문 열었더니 미세먼지 최악, 빨래 널었더니 오후 소나기. 미세먼지 앱은 "지금 나쁨"까지만 말하고 **"그래서 몇 시에 환기하라"**는 말하지 않습니다.

**케미:** 같은 4개 데이터 소스(시간별 대기질·습도·강수·자외선·일몰)를 집안일별로 **다른 수식**에 통과시켜 전혀 다른 판정을 뽑는 구조 — 환기 = PM2.5 저점 ∩ 강수 없음, 빨래·이불 = 저습도 ∩ 강수 0% ∩ 일조 ∩ 자외선(살균), 창문 열고 자기 = 밤 공기질 + 열대야. 조합 자체가 제품입니다.

- **핵심 API:** 기상청 단기예보(시간별 REH·POP·WSD 전부 제공 확인) + 에어코리아 + 기상청 생활기상지수(자외선) + Sunrise-Sunset(카탈로그, 무인증) + Telegram Bot
- **알려진 함정 1개(해소 가능):** '시간별 PM2.5 **예보**'는 정부 API에 없음(에어코리아는 실측, 예보통보는 일 단위 등급). → 카탈로그 안의 **OpenWeatherMap Air Pollution forecast**(시간별 4일, 무료) 또는 Open-Meteo Air Quality(키 불요)로 해결.
- **검증 결과:** 하루날씨("지금 빨래 가능" 한 문장), 에코캐스트(환기 단일), 케이웨더 빨래지수 등 인접 앱은 많지만 **다중 집안일 × 시간 단위 골든타임 × 선제 푸시** 조합은 비어 있음. 특히 이불 널기와 "소나기 전 빨래 걷어라" 역방향 알림은 아무도 안 함. 해자가 얕으니(공공 데이터) 빠른 실행 + 알림 품질로 승부.
- **MVP:** 집안일 3종 룰 엔진 + 시간 지정 푸시 + 타임라인 색칠 화면 1장. **1인 개발 2~3주 — 전체 목록에서 가장 빨리 검증 가능한 아이디어.**

## 4. 출발각 — 출퇴근 판정 앱

**BO 4 / FE 8 · 타겟: 수도권 지하철·버스 통근 직장인**

정보는 넘치는데 "그래서 지금 나가야 하나 / 오늘은 뭘 타야 하나"라는 **결론**은 아무 앱도 주지 않습니다. 캘린더 첫 일정에서 역산해 아침엔 "8:07 출발, 8:12 열차, 우산+마스크" 한 문장, 저녁엔 혼잡도×따릉이 잔여×강수를 합쳐 "오늘은 따릉이 30분 퇴근" 모드 추천.

- **핵심 API:** Google Calendar + Kakao Maps + City Bikes(이상 카탈로그) + 기상청 초단기예보 + 에어코리아 + 서울 지하철 실시간 도착(열린데이터광장) + 따릉이 실시간 대여정보
- **검증에서 잡힌 함정 2개:** ① TAGO 버스도착정보는 **서울 시내버스를 커버하지 않음** → 서울시 버스도착정보 API(api.bus.go.kr)로 교체 필수. ② '2호선 붐빔' 같은 노선·열차 혼잡도 오픈 API는 없음(서울시 citydata는 '장소' 혼잡도) → 문장을 "역삼역 일대 붐빔"으로 바꾸거나 SK오픈API 열차 혼잡도(무료체험) 추가.
- **경쟁 리스크가 점수를 깎음:** 에이닷 브리핑이 아침 종합 브리핑을 이미 제공, 카카오맵·네이버지도가 데이터 전부 보유. '판정 한 문장' UX의 차별화 수명이 짧을 수 있음. 다만 **왕복(아침+저녁) 의사결정 장악 + 정시출근 스트릭** 같은 리텐션 설계는 아직 아무도 없음.

## 5. 냉장고사이렌 — 식품 리콜 개인화 경보

**BO 4 / FE 8 · 타겟: 아이 있는 가정의 주 양육자, 요리하는 1인 가구**

식약처 회수·판매중지 발표는 매주 나오지만 "내가 산 그 제품"인지 대조할 방법이 없어 회수 식품 대부분이 그냥 소비됩니다.

**케미의 핵심 발견:** 식약처 회수 API(data.go.kr 15074318)가 **바코드번호 필드를 제공**합니다. 스캔해둔 냉장고 바코드 목록과 매일 밤 조인하면 전국 단위 리콜 공지가 "너의 냉장고 둘째 칸 그 소시지가 회수 대상"이라는 초개인화 경보로 바뀝니다. 국산품은 식약처 유통바코드 API, 수입품은 Open Food Facts/UPC database(카탈로그)로 식별 역할 분담.

- **검증에서 잡힌 함정:** EAN-13 바코드엔 날짜가 없어 '스캔만으로 D-day'는 불성립 → **바코드연계제품정보 C005**(품목별 '제조일로부터 N개월' 스펙 제공)로 소비기한을 자동 프리필하고 구매일만 확인받는 하이브리드 UX로 우회.
- **경쟁 현실:** 소비자24가 '관심품목 등록 → 리콜 시 알림'을 이미 무료 제공, 식약처 내손안이 바코드 1회 조회 제공. 남는 차별화는 **인벤토리 상시 감시 + 소비기한 관리 + 육아 특화 UX의 결합**. 푸드QR이 2026년 전 식품 확대되면 정부 기능으로 흡수될 리스크 있음 → 보험형 리텐션("지우면 우리집 감시가 꺼진다")을 빨리 쌓아야 함.

## 6. 댕냥레이더 — 산책 골든타임 + 실종 보험

**BO 3 / FE 9 (전 목록 실현성 1위, 치명 블로커 0건) · 타겟: 반려견 보호자 (연간 유실·유기 10만 마리)**

매일 쓰는 산책 기능과 절대 못 지우는 실종 보험이 한 앱에서 서로를 지탱하는 구조.

- **케미:** 산책 골든타임 = 기온(아스팔트 화상 추정)+미세먼지+자외선+일몰 4축 합성. 실종 레이더 = 품종·색·체중·지역 프로필로 **국가동물보호정보시스템 구조동물 API를 매시간 폴링**해 유사 공고 자동 매칭 푸시 → 보호센터 연락처·위치를 Kakao Maps로 즉시 연결.
- **실현성 근거:** 구조동물 API v2가 품종·색상·체중·특징·사진·보호소 연락처를 전부 제공하고 **개발계정 일 100만 콜**이라 전국 폴링도 여유. 자외선은 OpenUV(무료 50콜/일) 대신 기상청 생활기상지수로.
- **점수가 낮은 이유:** 산책 날씨는 포캐스트·다음날씨가, 사진 automatching은 '지켜줄개'가 이미 함. 조합 포지셔닝("평상시 산책앱 = 실종 시 보험")만 비어 있고 기술 장벽이 아니라 기획 차이라 방어력 낮음.

## 7. 관리비엑스레이 — 관리비 이상 탐지

**BO 3.5 / FE 8 · 타겟: 아파트 입주민 1,200만 세대, 입주자대표**

K-apt 데이터가 전부 공개돼 있지만 UX가 나빠 아무도 안 봅니다. 쌍둥이 단지(연식·세대수·난방방식 매칭) 대비 "경비비 18% 높음 = 세대당 연 14만원"이라는 협상 카드를 만들어주는 앱.

- **검증에서 확인:** 관리비는 **월별·항목별 오픈 API**(공용관리비 15057937 + 개별사용료 15059469)로 확보 가능, 기본정보 API에 난방방식·세대수·연식·부과면적 전부 실재. 기상청 ASOS로 난방도일 보정 성립.
- **점수가 낮은 이유:** K-apt 공식 기능에 '유사단지 1:N 비교'가 이미 있음. 민간 관리비알리미도 존재. 남는 것은 **날씨 보정 + 이상치 자동 감지 + "새는 돈 연 XX만원" 정량화 + 단톡방 공유용 카드** — 데이터가 아니라 UX·유통 싸움.
- **주의:** K-apt는 의무관리단지(약 1.8만)만 커버(빌라·오피스텔 불가), 공개 약 2개월 지연.

## 8. 살아보니 — 이사 리포트 + 입주 후 감시

**BO 4 / FE 7 · 타겟: 원룸·빌라·오피스텔 전월세를 구하는 20~30대**

- **살아남은 빈틈 2개:** ① **유흥업소 밀도**를 주거 관점에서 정량 리포트하는 서비스 부재('낮에 본 집이 밤엔 유흥가 한복판' 방지). ② **입주 후 내 집 반경 신규 건축 인허가·착공 상시 감시 + 푸시** — 계약 직후 옆 필지 공사가 시작되는 세입자의 최악 손실을 막아주는 소비자 서비스가 없음. 이사(저빈도)가 감시 구독(상시)으로 연착륙하는 리텐션 구조가 백미.
- **잘린 부분:** '계약 전 시세+치안 리포트' 절반은 직방 지킴진단(2026.4 출시)이 선점. 치안 점수화는 데이터상 불가(생활안전지도 범죄 데이터는 이미지 타일 전용) → '치안시설·안전 인프라 밀도 + 지도 오버레이'로 재정의 필요.
- **API 지뢰:** 인허가 데이터의 localdata.go.kr이 **2026-04-16 폐쇄됨** → 공공데이터포털로 이관된 동일 데이터 사용(좌표 포함, 오히려 유리).

## 9. 효도레이더 — 부모님 동네 원격 안심

**BO 2.5 / FE 8.5 · 만들기는 가장 쉬운 축에 속하나 블루오션은 아님**

재난문자·기상특보는 '그 지역 폰'에만 가서 자녀는 부모님 동네 폭염경보를 모릅니다. 특보를 "위험점수 78점 — 낮 외출 말라고 전화하세요"로 번역하고, 부모님 폰에는 아무것도 설치하지 않는 구조.

- **점수가 낮은 이유:** 안전디딤돌 앱의 '수신지역 설정'이 타지 재난문자 수신을 이미 제공하고, 기상청이 '폭염 영향예보 카카오톡 직접 전달'로 **"가족에게 안부 전화를 걸도록 돕는다"는 프레이밍까지 선점**. 남는 것은 통합 + 감성 카피 + 형제자매 분담 보드 정도의 얇은 UX 레이어.
- 그래도 만든다면: 기상특보 API(자동승인)만으로 먼저 출시하고 재난문자(safetydata.go.kr, 승인 수일)를 후속 추가하는 순서가 빠름. 카카오톡 '나에게 보내기'는 검수 없이 개인 개발자 사용 가능.

# 🧪 검증 과정에서 나온 공통 인사이트

**1. 해자는 API가 아니라 사용자가 쌓는 데이터에 있다.**
공공 API는 누구나(네이버·토스·정부 포함) 쓸 수 있어 조합 자체는 해자가 아닙니다. 살아남은 상위 아이디어의 공통점은 사용할수록 쌓이는 개인 데이터 — 코편한아침의 증상 로그, 냉장고사이렌의 인벤토리, 출발각의 통근 프로필 — 가 갈아타기 비용을 만든다는 것.

**2. 이기는 패턴은 '정보 나열'이 아니라 '판정 한 문장'.**
날씨·시세·혼잡도 데이터 앱은 전부 레드오션이지만, 결정의 순간(송금 직전·만기 D-7·퇴근 30분 전·아침 출발 전)에 **결론을 대신 내려주는** 앱은 대부분 비어 있었습니다. 여러 API를 하나의 수식에 태워야 결론이 나오기 때문에 — 그게 곧 케미입니다.

**3. '카테고리에 앱이 있다'와 '이 케미를 하는 앱이 있다'를 구분하라.**
18개 중 15개는 exists=false 판정이었지만 블루오션 점수는 대부분 2~6.5. 유사 카테고리 앱의 존재보다 **대기업·정부의 인접 확장 속도**가 진짜 변수였습니다(에이닷 브리핑, 푸드QR, 도시침수예보, 지킴진단).

**4. 검증 없이 기획하면 밟는 API 지뢰들 (이번 검증에서 실제로 발견):**

- 행안부 **localdata.go.kr 2026-04-16 폐쇄** → 공공데이터포털로 이관 (인허가 데이터 쓰는 모든 기획 영향)
- **TAGO 버스도착정보는 서울·경기 시내버스 미커버** → 서울시 api.bus.go.kr / 경기 GBIS 별도
- OpenWeatherMap **One Call 3.0은 카드 등록 필요** → 무료는 2.5 계열 + Air Pollution API
- OpenAQ **v1/v2 폐기, v3부터 API 키 필수** (카탈로그 auth 정보보다 최신)
- Domainsdb.info **무인증 아님**(구글 로그인 키로 전환), .kr 커버리지 사실상 없음
- 식약처 **식품영양성분 I2790은 ~2023 동결** → 신규 '식품영양성분DB정보'(15127578) 사용
- 재난문자 구 API 폐기 → **safetydata.go.kr 이관**(자동승인 아님, 수일 소요)
- 국세청 사업자 API는 **조회가 아니라 검증** — 번호만으로 업종·주소 못 얻음
- 심평원 비급여 API는 **병원급 이상만** — 의원급 없음
- 카카오맵 무료 쿼터 정책 **2026-07-21 변경**(계정당 첫 활성화 앱만 무료)
- 예금자보호 한도 **2025-09부터 1억 원** (5천만 원 기준 로직은 낡음)

**5. 카탈로그에서 '접착제'로 계속 불려나온 API들:**
